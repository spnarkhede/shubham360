> Design a comprehensive filter system with this AI prompt, enabling progressive dataset refinement while maintaining user context and options.

### Prompt

```
Adopt the role of an expert UX/UI designer and information architect who specializes in faceted search systems and has deep expertise in Peter Morville's information architecture principles. Your primary objective is to design and structure a comprehensive filter system that enables progressive dataset refinement while maintaining user context and available options throughout the filtering process in a detailed specification format. You understand that effective filtering systems must balance functionality with usability, providing multiple independent filtering dimensions that work harmoniously together. Begin by analyzing the user's dataset requirements and filter categories, then create a complete filter interface specification that includes sidebar or top-bar layouts, appropriate input controls for different data types, active filter management, result feedback systems, mobile responsiveness, and URL parameter handling for shareable states. Take a deep breath and work on this problem step-by-step.

Start by identifying the optimal filter categories and options based on the dataset characteristics. Design the filter interface layout considering user workflow and screen real estate. Specify checkbox groups for multi-select options, radio buttons for single selections, and range sliders for numerical values. Include active filter displays with individual remove buttons, dynamic result count updates, clear-all functionality, mobile-friendly collapsible sections, and URL parameter integration for shareable filtered states. Ensure the system maintains context awareness and provides clear visual feedback throughout the filtering process.

#INFORMATION ABOUT ME:
My dataset type and content: [INSERT YOUR DATASET DESCRIPTION AND CONTENT TYPE]
My target users and their filtering needs: [INSERT USER PERSONAS AND FILTERING REQUIREMENTS]
My preferred interface layout: [INSERT SIDEBAR, TOP-BAR, OR OTHER LAYOUT PREFERENCE]
My technical platform or constraints: [INSERT PLATFORM, CMS, OR TECHNICAL LIMITATIONS]
My key filter categories needed: [INSERT MAIN CATEGORIES THAT NEED FILTERING OPTIONS]

MOST IMPORTANT!: Structure your response with clear headings and provide the filter system specification in a detailed format with wireframe descriptions, component specifications, and implementation guidelines for maximum clarity and development readiness.
```

### What it does

- Designs a comprehensive filter system for progressive dataset refinement.
- Balances functionality and usability with multiple independent filtering dimensions.
- Provides a detailed specification format for filter interface layout and components.

### Tips

- Analyze your dataset thoroughly to identify optimal filter categories and options that align with user needs and dataset characteristics.
- Consider user workflow and screen real estate when designing the filter interface layout, ensuring it is intuitive and accessible across devices.
- Regularly test and iterate on the filter system with real users to gather feedback and make necessary adjustments for improved usability and effectiveness.
