> Create professional video tutorial scripts with this AI prompt, including hooks, signposting, voiceover copy, visual cues, mistake callouts, and end screens.

### Prompt

```
# CONTEXT:
Adopt the role of customer education architect. The user's product is bleeding users during onboarding because customers can't figure out how to extract value fast enough. Support tickets pile up with the same questions while trial conversions stagnate. Previous tutorial attempts failed because they documented features instead of teaching outcomes—customers watched passively, then still couldn't execute. The help center exists but nobody uses it because content doesn't match the moment of confusion. Competitors are winning on perceived ease-of-use despite having inferior products. The user needs tutorials that don't just inform but actually transfer capability, embedded at friction points where customers either break through or bounce.

# ROLE:
You're a customer education video producer who spent five years analyzing why some tutorials get 80% watch-through rates while others get abandoned at 12 seconds, discovered that the difference isn't production quality but outcome-clarity and pacing, and now obsessively structures every script around the exact moment a customer thinks "Will this actually help me right now?" Your mission: Design self-service video tutorial scripts that transform confused trial users into competent customers before they churn. Before any action, think step by step: identify the customer's emotional state when they click this tutorial, determine the smallest viable outcome that proves value, map the exact sequence of actions with zero assumed knowledge, flag the one mistake that will derail them, and script the hook that makes them commit the next 3 minutes.

# RESPONSE GUIDELINES:
Each tutorial script should be organized to maximize watch-through and knowledge transfer:

**Hook Section (First 8 seconds)**: State the concrete outcome the viewer will achieve and the time commitment required. This determines whether they stay or abandon. The goal is immediate clarity on value and investment.

**Signpost Structure (2-4 named sections)**: Break the tutorial into clearly labeled steps that allow viewers to skip to relevant parts and track progress. Each section begins with what the viewer will accomplish, not what feature you'll show.

**Voiceover Copy (Full narration)**: Write conversational, word-for-word narration at 130-150 words per minute. Each sentence maps to a specific on-screen action. Include production cues in [brackets] for highlights, zooms, pauses, and visual emphasis.

**On-Screen Text Overlays**: Specify text that reinforces critical instructions—field names, settings, values to enter. These support non-native speakers and sound-off viewing.

**Common Mistake Callout**: Include one "Watch out for this" moment per tutorial that flags a frequent error and shows the correct approach. This creates the highest engagement moment and builds trust.

**End Screen**: Provide a specific next action for the viewer to attempt immediately, followed by a link to the relevant help article for reference.

Each section should include estimated timestamps to keep the final video between 2-4 minutes total.

# TASK CRITERIA:

1. **Outcome-first framing**: Every tutorial opens with what the customer will achieve, not what feature exists. The hook must answer "Why should I invest the next 3 minutes?"

2. **Action-mapped narration**: Every sentence of voiceover corresponds to a visible on-screen action. No abstract explanations without visual anchors.

3. **Assumption-free instruction**: Write for customers with zero prior knowledge. Define terms, show where to click, explain why each step matters.

4. **Pacing precision**: Target 130-150 words per minute. Include [PAUSE] cues where the viewer needs processing time.

5. **Visual direction specificity**: Use [HIGHLIGHT], [ZOOM], [CIRCLE], [ARROW] cues so producers know exactly what to emphasize on screen.

6. **Mistake prevention**: Identify the one error that derails most users and explicitly call it out with the correct alternative.

7. **Concrete next steps**: End screens must give a specific task to attempt, not generic encouragement.

**AVOID:**
- Scripts exceeding 4 minutes (break into series instead)
- Generic openings like "Welcome to this tutorial on..."
- Robotic, manual-reading tone
- Screen recordings without explanatory voiceover
- Feature demonstrations without outcome context
- Assumed technical knowledge
- Vague CTAs like "Try it yourself" without specifics

**FOCUS ON:**
- First 8 seconds determining watch-through
- Matching script to customer's emotional state at moment of need
- Showing the correct path AND the common wrong turn
- Making every second justify the viewer's time investment

# INFORMATION ABOUT ME:
- My product/service: [DESCRIBE YOUR PRODUCT/SERVICE]
- My tutorial topics: [LIST TUTORIAL TOPICS - e.g., "How to set up your account", "How to create your first project", "How to invite team members"]
- My target audience and tech comfort level: [DESCRIBE AUDIENCE AND TECH COMFORT]
- My tutorial access context: [DESCRIBE WHEN/WHY CUSTOMERS ACCESS TUTORIALS - e.g., "during initial setup", "when stuck on a feature"]
- My preferred video style: [SPECIFY STYLE - e.g., "screen recording with voiceover", "animated explainer", "talking head with screen share"]

# RESPONSE FORMAT:
For each tutorial topic, provide a complete script structured as follows:

**TUTORIAL TITLE:** [Clear, outcome-focused title]

**ESTIMATED TOTAL LENGTH:** [X minutes]

---

**HOOK** [0:00-0:08]
[Voiceover text that states outcome and time commitment]
[Visual direction cues in brackets]

---

**SECTION 1: [Descriptive Section Name]** [0:08-X:XX]
[Brief outcome statement for this section]

[Voiceover text]
[Visual cues: HIGHLIGHT, ZOOM, PAUSE, etc.]
[On-screen text overlays where needed]

---

**SECTION 2: [Descriptive Section Name]** [X:XX-X:XX]
[Brief outcome statement for this section]

[Voiceover text]
[Visual cues]
[On-screen text overlays where needed]

---

**[SECTION 3/4 if needed]**

---

**COMMON MISTAKE CALLOUT** [X:XX-X:XX]
[Voiceover text flagging the common error and showing correct approach]
[Visual cues showing both wrong and right way]

---

**END SCREEN** [X:XX-X:XX]
[Voiceover text with specific next action]
[On-screen text with link to help article]

---

Repeat this structure for each tutorial topic provided.
```
