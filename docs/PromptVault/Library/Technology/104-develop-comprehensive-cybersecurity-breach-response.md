> Develop a comprehensive, step-by-step cybersecurity breach response plan for large enterprises, incorporating real-world examples and case studies

### Prompt

```
Develop a comprehensive, step-by-step cybersecurity breach response plan for large enterprises, incorporating real-world examples and case studies. Tailor the plan for advanced users with in-depth knowledge of cybersecurity threats and mitigation techniques.** The plan should cover:** ## 1. **Threat Assessment**: Outline the process for identifying and analyzing the breach, including tools and methodologies for threat detection and classification. ## 2. **Containment**: Describe strategies for isolating affected systems and preventing further damage, detailing specific containment techniques and their applications. ## 3. **Eradication**: Provide a step-by-step guide on removing the threat, including procedures for system restoration and validation. ## 4. **Recovery**: Explain the process for restoring systems and services to normal operation, highlighting key considerations for data integrity and business continuity. ## 5. **Post-Incident Activities**: Outline procedures for conducting a thorough post-breach analysis, including lessons learned, incident reporting, and future prevention strategies. Ensure each step is clearly explained with technical rationale and real-world examples, reflecting an advanced understanding of cybersecurity threats and mitigation techniques.
```

### What it does

- Provides a methodical roadmap for identifying and mitigating security risks.
- Covers threat modeling, zero-trust principles, and incident response.
- Aligns implementation with industry standards such as NIST, ISO 27001, and OWASP.

### Tips

- Describe your current security posture and existing controls for a tailored roadmap.
- Reference specific compliance frameworks (NIST, ISO 27001) if they apply to your organization.
- Ask for a phased implementation plan to make the roadmap actionable.

### How to use

1. Use Claude or ChatGPT for this prompt — both handle security nuance well.
2. Describe your organization size, industry, and existing security tools.
3. Work through the roadmap phase by phase, asking for specifics on tools and configurations.
4. Request a checklist or table format for easier tracking of implementation tasks.
