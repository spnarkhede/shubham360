### Prompt

```
Develop a comprehensive, step-by-step guide for migrating to quantum-resistant cryptography, targeting advanced users.** Include:** ## 1. **Risk Assessment**: Outline methodologies for identifying vulnerabilities and quantifying the risk of quantum attacks on existing cryptographic systems. ## 2. **Algorithm Selection**: Discuss criteria for choosing suitable quantum-resistant algorithms, considering factors such as security levels, performance, and compatibility. ## 3. **Implementation Considerations**: Provide technical guidance on integrating quantum-resistant cryptography into existing systems, including key management, protocol updates, and backward compatibility. ## 4. **Overcoming Common Challenges**: Offer actionable advice and real-world examples for addressing typical obstacles encountered during migration, such as interoperability issues and performance optimization. ## 5. **Case Studies**: Include detailed, technical case studies of successful migrations to quantum-resistant cryptography, highlighting lessons learned and best practices. Ensure the guide is tailored to sophisticated users, maintains a technical tone, and focuses on providing in-depth strategies for securing systems against quantum threats.
```

### What it does

- Provides a methodical roadmap for identifying and mitigating security risks.
- Covers threat modeling, zero-trust principles, and incident response.
- Aligns implementation with industry standards such as NIST, ISO 27001, and OWASP.

### Tips

- Describe your current security posture and existing controls for a tailored roadmap.
- Reference specific compliance frameworks (NIST, ISO 27001) if they apply to your organization.
- Ask for a phased implementation plan to make the roadmap actionable.

### How to use

1. Use Claude or ChatGPT for this prompt — both handle security nuance well.
2. Describe your organization size, industry, and existing security tools.
3. Work through the roadmap phase by phase, asking for specifics on tools and configurations.
4. Request a checklist or table format for easier tracking of implementation tasks.
