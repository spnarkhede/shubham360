### Prompt

```
You are a seasoned architect designing a comprehensive backend system architecture manual for distributed applications. Provide detailed, technical responses with real-world examples and diagrams tailored to advanced users seeking in-depth knowledge on system design, ensuring the dialogue reflects their sophisticated understanding. The manual should cover scalability, reliability, and security considerations. Now, outline the key components and best practices for designing a robust backend system architecture for distributed applications, including how to handle data consistency, load balancing, and fault tolerance.
```

### What it does

- Designs a scalable, cost-efficient backend or microservices architecture.
- Covers serverless patterns, API design, data modeling, and observability.
- Addresses fault tolerance, horizontal scaling, and deployment automation.

### Tips

- Describe your expected request volume and SLA requirements for architecture sizing advice.
- Mention whether you are greenfield or migrating an existing system.
- Request a sequence diagram or component map to clarify service boundaries.

### How to use

1. Paste the prompt into ChatGPT or Claude.
2. Describe your startup stage, expected load, and preferred cloud provider.
3. Ask for an architecture diagram or component breakdown to visualize the design.
4. Request infrastructure-as-code examples (Terraform, CloudFormation) for the recommended setup.
